# `.baerly/schema.lock.json` is shipped trivially empty

**Severity: LOW. Minor scaffold cleanliness; not user-facing.**

Each scaffoldable template ships an identical
`.baerly/schema.lock.json`:

```json
{
  "schema_version": 1,
  "comment": "...",
  "tables": {}
}
```

(`tables: {}` in all three —
`examples/minimal-cloudflare/.baerly/schema.lock.json`,
`examples/minimal-node/.baerly/schema.lock.json`,
`examples/helpdesk-cloudflare/.baerly/schema.lock.json`.)

`AGENTS.md` in each template states schema.lock is advisory ("an
empty `{tables: {}}` is fine when you supply schemas in code"). So
the file is *informationally* empty but *positionally* prime — it
sits in `.baerly/` alongside `scaffold.json`, suggesting it's
load-bearing infrastructure.

Two paths, both small:

## Option A — Drop from the scaffold

Delete the file from each template's `.baerly/` dir. Document in
`AGENTS.md` that "schema-lock is generated by `baerly migrate
--write-lock`; if you're declaring schemas in code, you don't
need a lockfile." User runs the relevant command if they want one.

**Smallest scaffolded surface area.**

## Option B — Include one tiny example entry

Replace `tables: {}` with one example table entry plus a comment
showing the shape:

```json
{
  "schema_version": 1,
  "comment": "Pin column shapes here. Remove this template entry — it's only a shape hint.",
  "tables": {
    "example": {
      "columns": { "id": "string", "created_at": "string" }
    }
  }
}
```

The user removes the example entry when they're ready. They've
seen the shape; they know the file isn't load-bearing-empty.

**Preserves discoverability of the lockfile feature.**

## Recommendation

Option A. The user-facing API today is `defineConfig({ collections:
{ tickets: { schema, indexes } } })` — schemas in code. The lockfile
is an audit/migration aid, not a day-1 affordance. Don't pre-stuff a
scaffold with a near-empty audit artifact.

If someone files a feature request asking "where's my schema
lockfile?", revisit with Option B and add the example entry then.
