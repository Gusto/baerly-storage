# Deprecated path tokens — audit trail for renames.
#
# When you rename a file or change a layout convention, add the OLD
# path/token here in the same commit. `pnpm check:references` (run by
# lefthook pre-commit) then fails any later commit that reintroduces it.
#
# Format: one token per line. Inline `#` starts a comment. Tokens are
# matched as fixed substrings (regex metachars escaped).
#
# Example:
#   __tests__/                  # 2026-05-09: tests moved to colocated + tests/
#   src/operationQueue.ts       # 2026-05-09: kebab-cased to operation-queue.ts
#
# NOTE: file-move tokens whose OLD path is a substring of the NEW path
# (e.g. `src/types.ts` is a substring of `packages/protocol/src/types.ts`)
# would false-positive on the moved files themselves. Skip those —
# typecheck catches the relevant regression: any new `from "./types"`
# (etc.) in `src/` fails to resolve since the file no longer exists
# there. The entries below are deletion-only or renamed-with-no-substring.
